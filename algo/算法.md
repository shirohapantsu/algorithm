# 算法

## 1.枚举 (Emunerate)

枚举算法（Enumeration Algorithm），又称穷举算法，是指根据问题的特点，逐一列出所有可能的解，并与目标条件进行比较，找出满足要求的答案。枚举时要确保不遗漏、不重复。

枚举算法的核心思想就是：遍历所有可能的状态，逐个判断是否满足条件，找到符合要求的解。

**枚举 = 穷举所有可能 + 判断是否符合要求**

### 基本步骤

1. 明确需要枚举的对象、枚举范围和约束条件。
2. 逐一枚举所有可能情况，判断是否满足题意。
3. 思考如何提升枚举效率。

例题：[P1089 [NOIP 2004 提高组] 津津的储蓄计划](https://www.luogu.com.cn/problem/P1089)

## 3.递归 (Recursion)

递归（Recursion）：是一种将复杂问题分解为与原问题结构相同的子问题，并通过重复求解这些子问题来获得最终解答的方法。在大多数编程语言中，递归通常通过函数自身的调用来实现。

简而言之，递归包含「递推过程」和「回归过程」：

- 递推过程：将大问题逐步分解为更小的同类子问题，直到终止条件。
- 回归过程：从最小子问题开始，逐层返回结果，最终解决原问题。

### 基本方法
递归的核心思想是：把大问题拆解为小问题，逐步解决。写递归时，可以遵循以下三步：

1. 写递推公式：找出原问题与子问题的关系，写出递推公式。
2. 确定终止条件：明确递归何时结束，以及结束时的返回值。
3. 翻译为代码：
   - 定义递归函数（明确参数和返回值含义）
   - 编写递归主体（递推公式对应的递归调用）
   - 加入终止条件的判断和处理

写递归的要点：
>明白一个函数的作用并相信它能完成这个任务，千万不要跳进这个函数里面企图探究更多细节， 否则就会陷入无穷的细节无法自拔，人脑能压几个栈啊．

例题： [0070.爬楼梯-leetcode](https://leetcode.cn/problems/climbing-stairs/)

### 递归优化

递归算法常常会遇到重复计算的问题，为避免重复运算，可以引入缓存机制（如哈希表、数组或集合）记录已经计算过的子问题结果。这种做法称为「记忆化递归」或「递归 + 备忘录」，也是动态规划的核心思想之一。每次递归调用前，先检查缓存中是否已有结果，如果有则直接返回，无需再次递归，从而显著提升效率。

## 4.分治（Divide and Conquer）

分治算法（Divide and Conquer）：即「分而治之」，把一个复杂问题拆分成多个相同或相似的子问题，递归分解，直到子问题足够简单可以直接解决，最后将子问题的解合并得到原问题的解。

分治和递归都强调「拆分问题」。递归是一种实现方式，分治是一种思想。可以理解为：递归算法⊂分治算法递归算法⊂分治算法。

![归并排序的分治策略](https://www.hello-algo.com/chapter_divide_and_conquer/divide_and_conquer.assets/divide_and_conquer_merge_sort.png)

### 基本方法
大概的流程可以分为三步：分解 -> 解决 -> 合并．

1. 分解原问题为结构相同的子问题．
2. 分解到某个容易求解的边界之后，进行递归求解．
3. 将子问题的解合并成原问题的解．

例题：[P1010 [NOIP 1998 普及组] 幂次方](https://www.luogu.com.cn/problem/P1010)

### 二分查找

**二分查找**(binary search)，又称折半查询、对数搜索，是用来在一个**有序数组**中查找**某一**特定元素的算法

从定义中我们可以提取出使用二分查找的条件：
- 被查找的数组必须有序
- 查找的对象只能是一个

在二分查找中，必须要注意查找范围的左右区间开闭问题，也就是查找时的边界问题，具体分为：
- 左闭右闭 `[left,right]`
- 左闭右开 `[left,right)`

这决定了二分法中最重要的两个点：
- while循环条件到底是 `left <= right` 还是 `left < right`
- 迭代过程中构造下一次迭代的区间边界时到底要不到移一位(就是 `right = mid` 还是`right = mid -1`的问题)

#### 左闭右闭 `[left,right]`

代码范式：
```cpp
int bin_search(vector<int> nums,int target)
{
    int left = 0;
    int right = nums.size() - 1;    //左闭右闭

    while(left <= right)    //因为左闭右闭，所以left == right是有效的
    {
        int mid =left + (right - left) / 2; //不写 mid = (left+right)/2 防止整数溢出
        if(num[mid] > target) right = mid - 1;
        else if(nums[mid] < target) left = mid +1;
        // 由于双闭带来的left == right的有效，在定义下一次迭代的边界条件时一定要保证迭代范围的严格缩小，防止向下取整和左右边界相等时带来的死循环
        else return mid;
    }
    return -1;
}
```

要点：
- 初始化：`right = n - 1`
- 循环条件：`while(left <= right)`
- 边界更新：`right = mid - 1` `left = mid + 1`

#### 左闭右开`[left ,right)`

代码范式：
```cpp
int bin_search(vector<int> nums;int target)
{
    int left = 0;
    int right = nums.size();    //左闭右开

    while(left < right)     //left == right时区间内只有一个元素，无再次比较的意义
    {
        int mid = left + (right - left) / 2;
        if(nums[mid] > target) right = mid;
        else if(nums[mid] < target) left = mid + 1;
        // 这里因为不存在left == right的情况了，所以对于右边界来说不用严格缩小也不会进入四循环；但是左边界的整除向下取整带来的可能死循环依旧没能解决，所以还是得+1
        else return mid;
    } 
    return -1;
}
```

要点：
- 初始化：`right = n`
- 循环条件：`while(left < right)`
- 边界更新：`right = mid` `left = mid + 1`

#### 总结：有闭就一,两闭加等
  
## 5.回溯 (Backtracking)

回溯算法（Backtracking）：回溯算法是一种系统地搜索所有可能解的算法，通过递归和试错的方式逐步构建解的过程。当发现当前路径无法满足题目要求或无法得到有效解时，算法会撤销上一步的选择（即「回溯」），返回到上一个决策点，尝试其他可能的路径。回溯法的核心思想是「走不通就退回，换条路再试」，而每次需要回退的节点称为「回溯点」。

回溯的本质上是一种深度优先搜索算法,是在遍历一颗搜索树，BST的遍历就是回溯算法

### 剪枝

剪枝的思想就是：当发现某条搜索路径不可能得到正确答案时，就提前停止继续搜索；其本质就是利用题目的约束条件来减少搜索

### 基本方法
1. 明确所有选择：画出决策树，理清每一步有哪些可选项。每个节点的分支代表一次选择。
2. 明确终止条件：终止条件通常是递归到某一深度、遍历完所有元素或满足题目要求。到达终止条件时，处理当前结果（如加入答案集）。
3. 将决策树和终止条件转化为代码：
   - 定义回溯函数（明确函数意义、传入参数、返回结果等）。
   - 书写回溯函数主体（给出约束条件、选择元素、递归搜索、撤销选择部分）。
   - 明确递归终止条件（给出递归终止条件，以及递归终止时的处理方法）。

### 回溯算法的代码框架

___尝试、回退、剪枝___
```cpp
void backtrack(State *state, vector<Choice*> &choices, vectoe<State*> &res)
{
    // 判断是否为解
    if(isSolution(state))
    {
        // 记录解
        recordSolution(state,res);
        // 不再继续搜索
        return;
    }

    //遍历所有选择
    for(Choice choice: choices)
    {
        // 剪枝并判断是否合法
        if(isValid(state,choice))
        {
            //尝试：作出选择，更新状态
            makeChoice(state,choice);
            backtrack(state,chioces,res);
            // 回退：撤销选择，恢复到之前的状态
            undoChoice(state,choice);
        }
    }
}
```       
在这个代码框架下，，回溯算法可以说被简化为了一道填空题
- state：当前问题的状态
- choice：当前状态下可以作出的选择
- res：问题的结果的集合

- 约束满足问题：[n皇后](https://leetcode.cn/problems/n-queens/)

## 6.排序 (sorting)

___本部分所提到的算法默认使用数组实现___

排序算法（sorting algorithm）用于对一组数据按照特定顺序进行排列；排序算法中的数据类型可以是整数、浮点数、字符或字符串等。排序的判断规则可根据需求设定，如数字大小、字符 ASCII 码顺序或自定义规则。

排序的**稳定性**：稳定排序在完成排序后，相等元素在数组中的相对顺序不发生改变。

### 选择排序

选择排序（英语：Selection sort）是一种简单直观的排序算法．它的工作原理是每次找出第 $i$ 小的元素（也就是 $A_{i..n}$ 中最小的元素），然后将这个元素与数组第 $i$ 个位置上的元素交换．

- 不具有稳定性
- 时间复杂度为 $O(n^2)$

### 冒泡排序

冒泡排序（英语：Bubble sort）的工作原理是每次检查相邻两个元素，如果前面的元素与后面的元素满足给定的排序条件，就将相邻两个元素交换．当没有相邻的元素需要交换时，排序就完成了．

- 具有稳定性
- 时间复杂度为 $O(n^2)$ 

### 插入排序

插入排序（insertion sort）的工作原理为将待排列元素划分为「已排序」和「未排序」两部分，每次从「未排序的」元素中选择一个插入到「已排序的」元素中的正确位置．

- 具有稳定性
- 时间复杂度为 $O(n^2)$

在小数据量的情况下，时间复杂度为 $O(n^2)$ 的插入排序反而快于时间复杂度为 $O(n\log n)$ 的快速排序，因为快排基于分治，单元操作数更大；在数据量较小的情况下，两者时间复杂度拉不开差距，但是插排的单元操作数更小

插入排序优于冒泡和选择：
- 单元操作量更低（仅需赋值而不涉及swap）
- 部分有序时效率高于选择
- 具有稳定性
 
### 希尔排序
希尔排序（Shell Sort）基本思想：通过设定不同的间隔（gap），将数组分组进行插入排序，然后逐步缩小间隔直至为1，最终完成整个数组的排序。

- 不具有稳定性
- 时间复杂度为 $O(n)$ ~ $O(n^2)$

### 快速排序

快速排序（Quick Sort）基本思想：采用分治策略，选择一个基准元素，将数组分为两部分：小于基准的元素放在左侧，大于基准的元素放在右侧。然后递归地对左右两部分进行排序，最终得到有序数组。

- 不具有稳定性
- 时间复杂度为 $O(n\log n)$

### 归并排序

归并排序（Merge Sort）基本思想：利用分治法，将数组递归地一分为二，直至每个子数组只包含一个元素。随后，将这些有序子数组两两合并，最终得到一个整体有序的数组。

- 具有稳定性
- 时间复杂度为 $O(n\log n)$
- 不是原地排序（需要借助额外的数组）

### 堆排序

堆排序（Heap sort）基本思想：利用堆的特性，将数组构建成大顶堆，然后重复取出堆顶元素（最大值）并调整堆结构，最终得到有序数组。

- 不具有稳定性
- 时间复杂度为 $O(n \log n)$

### 桶排序

桶排序（Bucket Sort）：将待排序元素分散到多个桶中，对每个桶单独排序后合并。

将原数组中的元素按照一定规则映射到每个桶中，一般可以按照`(值 - 最小值) / 桶大小`进行映射，此时桶与桶之间有序，只需要对桶内进行排序后合并即可

- 稳定性取决于桶内排序使用的算法
- 时间复杂度为 $O(n+k)$
- 需要额外的空间

### 计数排序

计数排序（Counting Sort）基本思想：统计数组中每个元素出现的次数，然后根据统计信息将元素按顺序放置到正确位置，实现排序。

实际上可以看作桶排序的一种bucket_size=1的特例

- 具有稳定性
- 时间复杂度为 $O(n+k)$

### 基数排序

基数排序（英语：Radix sort）是一种非比较型的排序算法，最早用于解决卡片排序的问题．基数排序将待排序的元素拆分为 $k$ 个关键字，逐一对各个关键字排序后完成对所有元素的排序．

如果是从第 1 关键字到第 $k$ 关键字顺序进行比较，则该基数排序称为 MSD（Most Significant Digit first）基数排序；

如果是从第 $k$ 关键字到第 1 关键字顺序进行比较，则该基数排序称为 LSD（Least Significant Digit first）基数排序．

## 7.贪心 (greedy algorithm)

贪心算法（Greedy Algorithm）：每一步都选择当前最优（看起来最好的）方案，期望通过一系列局部最优，最终获得全局最优解。

贪心算法适用于一类特殊问题：只要每一步都做出当前最优选择，最终就能得到整体最优解或近似最优解。但并非所有问题都适用贪心算法。

通常，能用贪心算法解决的问题需同时满足两个条件：
- 贪心选择性质：全局最优解可以通过一系列局部最优（贪心）选择获得。
- 最优子结构：问题的最优解包含其子问题的最优解。

### 基本方法
贪心问题的解决流程大体可分为以下三步。

- 问题分析：梳理与理解问题特性，包括状态定义、优化目标和约束条件等。这一步在回溯和动态规划中都有涉及。
- 确定贪心策略：确定如何在每一步中做出贪心选择。这个策略能够在每一步减小问题的规模，并最终解决整个问题。
- 正确性证明：通常需要证明问题具有贪心选择性质和最优子结构。这个步骤可能需要用到数学证明，例如归纳法或反证法等。

## 8.搜索 (Search)

### 深度优先搜索DFS (depth first search)

每次尝试时都向更深的节点走

基于递归实现，回溯就是一种特殊的DFS，回溯 = DFS + 剪枝 + 撤销选择

### 广度优先搜索BFS (breadth first search)

每次都尝试访问同一层的节点，如果同一层都访问完了，再访问下一层
这样做的结果是：BFS算法找到的路径是从起点开始的 **最短合法路径**，也就是说这条路包含的边数最小，所以适合用来求解**最短路径**或**最少步骤**类的问题

### 比较
DFS和BFS的搜索，DFS本质上是牺牲了时间负责读换来了较低的空间复杂度；BFS是牺牲了空间复杂度换来了较低时世家复杂度

## 9.动态规划 (Dynamic Programming)

动态规划（Dynamic Programming，DP）：是一种求解多阶段决策过程最优化问题的方法。在动态规划中，通过把原问题分解为相对简单的子问题，先求解子问题，再由子问题的解而得到原问题的解。

动态规划的核心思想：

1. 把「原问题」分解为「若干个重叠的子问题」，每个子问题的求解过程都构成一个「阶段」。在完成一个阶段的计算之后，动态规划方法才会执行下一个阶段的计算。
2. 在求解子问题的过程中，按照「自顶向下的记忆化搜索方法」或者「自底向上的递推方法」求解出「子问题的解」，把结果存储在表格中，当需要再次求解此子问题时，直接从表格中查询该子问题的解，从而避免了大量的重复计算。

能使用dp解决的问题，需要满足三个条件：**最优子结构**、**无后效性**和**子问题重叠**

1. **最优子结构**：证明问题最优解的第一个组成部分是做出一个选择，假定你已经知道哪种选择才会得到最优解，确定这次选择会产生哪些子问题，以及如何刻画子问题空间，反证法证明子问题的解必须是它本身的最优解
2. **无后效性**：已经求解的子问题，不会再受到后续决策的影响
3. **子问题重叠**：如果有大量的重叠子问题，我们可以用空间将这些子问题的解存储下来，避免重复求解相同的子问题，从而提升效率

> 关于最优子结构的说明：
>
> [IOI 1994] 数字三角形：给定一个由数字组成的三角形，每次只能向左下或右下走，求从顶点到底部的路径中，经过的数字之和最大的一条路径。
> 1. 第一步：证明问题最优解的第一个组成部分是做出一个选择：要构成从顶点 $(1, 1)$ 到最后一行的最大路径和（最优解），我们在任何一个节点 $(i, j)$ 都必须做出一个选择：下一步到底是往左下 $(i+1, j)$ 走，还是往右下 $(i+1, j+1)$ 走？最优解的路径中，必然包含了在这个节点上做出的某一个具体选择。
> 2. 第二步：假定你已经知道哪种选择才会得到最优解：现在，我们假设有一个“上帝视角”，它告诉我们：对于当前节点 $(i, j)$，最优路径是向左下走（即走到 $(i+1, j)$）。我们不需要关心这个上帝视角是通过什么算法算出这个结论的，只假定这个选择已经是板上钉钉的最优决定。
> 3. 第三步：确定这次选择会产生哪些子问题，以及如何刻画子问题空间：因为我们已经做出了选择——从 $(i, j)$ 走向了 $(i+1, j)$，那么接下来的问题就变成了：如何从 $(i+1, j)$ 这个节点出发，走到三角形底部，使得路径和最大？子问题：求从 $(i+1, j)$ 到底部的最大路径和。子问题空间刻画：我们可以用坐标 $(x, y)$ 来完美刻画所有的子问题状态，表示“从第 $x$ 行第 $y$ 列出发到达底部的最大路径和”，记为 $f(x, y)$。
> 4. 第四步：反证法证明子问题的解必须是它本身的最优解：我们假设当前从 $(1, 1)$ 出发的总最优解路径，经过了 $(i, j)$ 并选择走向了 $(i+1, j)$。假设：这条总路径中，从 $(i+1, j)$ 到底部的那部分子路径（设其和为 $S_{sub}$）并不是 $(i+1, j)$ 到底部的最优解。推论：这意味着，存在另一条从 $(i+1, j)$ 到底部的路径，其和为 $S'_{sub}$，且 $S'_{sub} > S_{sub}$。矛盾：如果是这样，我们完全可以在原问题的总路径中，把从 $(i+1, j)$ 开始的后半段“剪掉”，拼接上和更大的 $S'_{sub}$ 路径。这样一来，总的路径和就会变得更大，这就推翻了我们在开头“总路径是最优解”的假设。结论：矛盾成立！因此，原问题最优解中所包含的子问题解（从 $(i+1, j)$ 到底部），必须是该子问题的最优解。这就是最优子结构。

#### 术语

##### 一、 状态 (State)

*   **大白话**：对问题在某一时刻所处“阶段”的**精确数学描述**。
*   **本质**：它是解题过程中的一个“快照”，用来严格定义子问题到底是什么。我们通常用数组（一维 `dp[i]` 或二维 `dp[i][j]`）或者函数的参数（`dfs(i, j)`）来表示状态。
*   **举例**：
    *   **台阶问题**：状态定义为 $dp[i]$，表示“爬到第 $i$ 级台阶的最小花费”。
    *   **数字三角形**：状态定义为 $dp[i][j]$，表示“从起点走到第 $i$ 行第 $j$ 列的最大路径和”。
*   **核心准则**：状态定义必须满足“无后效性”（过去不决定未来，未来不改变过去）。

---

##### 二、 状态转移方程 (State Transition Equation)

*   **大白话**：状态与状态之间的**进化规则**。也就是当前这个状态，是由历史上的哪些状态推导过来的。
*   **本质**：一个核心的数学递推式。它描述了我们如何在子问题的状态空间图（DAG，有向无环图）上进行计算的依赖关系。
*   **举例**：
    *   **台阶问题**：$dp[i] = \min(dp[i-1], dp[i-2]) + cost[i]$
    *   **数字三角形**：$dp[i][j] = \max(dp[i-1][j], dp[i-1][j-1]) + value[i][j]$

---

##### 三、 边界条件 / 初始状态 (Base Case / Boundary Condition)

*   **大白话**：推导的**绝对起点**。那些小到不能再小、不需要任何公式计算就能直接得出固定答案的子问题。
*   **本质**：它是记忆化搜索（递归树）的叶子节点终止条件，也是递推型 DP（for 循环填表）最开始必须手动赋初值的那几个元素。如果没有边界条件，状态转移方程就会陷入死循环或数组越界。
*   **举例**：
    *   **台阶问题**：站在平地（起点）不需要花费体力，因此边界条件通常是 $dp[0] = 0, dp[1] = 0$。

#### 基本方法

1. 划分阶段，确定**状态**，得到dp表
2. 根据决策方法，确定**最优子结构**，得到**状态转移方程**
3. 根据边界条件，完成dp

### 记忆化搜索 (Memorization Search)

记忆化搜索（Memoization Search）：是一种通过存储已经遍历过的状态信息，从而避免对同一状态重复遍历的搜索算法。

记忆化搜索本质上是动态规划“自顶向下”的递归实现，也可以看作是一个带缓存的DFS；常规的DP是靠递推公式实现的

从图论的视角来看，DFS是在遍历一棵树，而记忆化搜索则是将树中每条路的重复部分进行折叠，折叠形成了一个有向无环图(DAG)

#### 基本方法

我们在使用记忆化搜索解决问题的时候，其基本步骤如下：

1. 写出问题的动态规划「状态」和「状态转移方程」。
2. 定义一个缓存（数组或哈希表），用于保存子问题的解。
3. 定义一个递归函数，用于解决问题。在递归函数中，首先检查缓存中是否已经存在需要计算的结果，如果存在则直接返回结果，否则进行计算，并将结果存储到缓存中，再返回结果。
4. 在主函数中，调用递归函数并返回结果。

### 背包问题

>背包问题：背包问题是线性 DP 问题中一类经典模型。其基本描述为：给定若干物品，每种物品有各自的重量、价值和数量限制，以及一个最大承重为 W 的背包。要求在不超过背包承重上限的前提下，选择若干物品放入背包，使得背包内物品的总价值最大。

#### 0-1背包问题

>给定 $n$ 件物品和一个最大承重为 $W$ 的背包。每件物品的重量为 $weight[i]$ ，价值为 $value[i]$ ，每种物品只能选择一次。问在不超过背包承重的前提下，最大能获得多少总价值

1. 阶段：以物品序号及背包剩余容量为阶段  
2. 状态：$dp[i][j]$ 表示在前 $i$ 件物品时，背包剩余容量为 $j$ 的情况下，背包中的总价值  
3. 状态转移方程：$dp[i][j] = \begin{cases} dp[i-1][j]  \\ dp[i-1][j+weight[i]]+value[i] \end{cases}$ ， 在当前语境下即 $dp[i][j] = \max(dp[i-1][j],dp[i-1][j+weight[i]]+value[i])$
4. 初始条件：$dp[i][0]=0$ ， $dp[0][j] = 0$
5. 问题结果：$dp[n][W]$

如果dp的每一阶段只依赖于上一阶段，即不涉及再以前的阶段时，可以使用滚动数组优化空间复杂度：
- 状态转移方程：$dp[j]=\max(dp[j],dp[j-weight[i]]+value[i])$
- 遍历方向：从大到小倒序遍历； $dp[j-weight[i]]$ 是 $dp[j]$ 左边的数据，从右往左遍历可以保证取到的是上一轮中的数据，以此确保不会拿到重复的物品

```cpp
// 二维朴素0-1背包dp
#include <bits/stdc++.h>
using namespace std;

int n,W;    // n件物品和背包容量W
int weight[100];    // 每件物品的重量数组
int value[100];     // 每件物品的价值数组
int dp[100][100] =  {0};    // 二维dp表,这里的初始化隐含了边界条件

int main()
{
    cin>>n>>W;
    for(int i=1;i<=n;i++) cin>>weight[i]>>value[i];

    for(int i=1;i<=n;i++)
        for(int j=1;j<=W;j++)
            {
                if(j<weight[i]) dp[i][j] = dp[i-1][j];
                else dp[i][j] = max(dp[i-1][j],dp[i-1][j-weight[i]]+value[i]);
            }
    
    cout << dp[n][W];
    return 0;
}

```
时间复杂度：$O(nW)$，空间复杂度：$O(nW)$。

#### 完全背包问题

>给定 $n$ 种物品和一个最大承重为 $W$ 的背包。每种物品的重量为 $weight[i]$ ，价值为 $value[i]$ ，每种物品都有无限件可用。问在不超过背包承重的前提下，最多能获得多少总价值

1. 阶段：以物品序号及背包当前容量为阶段  
2. 状态：$dp[i][j]$ 表示在前 $i$ 种物品中进行选择时，背包容量为 $j$ 的情况下，背包中的最大总价值  
3. 状态转移方程：$dp[i][j] = \begin{cases} dp[i-1][j]  \\ dp[i][j-weight[i]]+value[i] \end{cases}$ ， 在当前语境下即 $dp[i][j] = \max(dp[i-1][j], dp[i][j-weight[i]]+value[i])$
4. 初始条件：$dp[i][0]=0$ ， $dp[0][j] = 0$
5. 问题结果：$dp[n][W]$

由于完全背包的二维状态转移方程中，当前状态 $dp[i][j]$ 仅依赖于上一阶段的 $dp[i-1][j]$ 和当前阶段较小容量的 $dp[i][j-weight[i]]$，同样可以使用一维滚动数组极大地优化空间复杂度：
- 状态转移方程：$dp[j] = \max(dp[j], dp[j-weight[i]]+value[i])$
- 遍历方向：从小到大**正序遍历**，以确保计算 $dp[j]$ 时调用的 $dp[j-weight[i]]$ 是本轮已经更新过的新数据，以此实现“同一种物品可以被放入多次”

```cpp
// 二维朴素完全背包dp
#include <bits/stdc++.h>
using namespace std;

int n, W;           // n件物品和背包容量W
int weight[100];    // 每件物品的重量数组
int value[100];     // 每件物品的价值数组
int dp[100][100] = {0}; // 二维dp表,这里的初始化隐含了边界条件

int main()
{
    cin >> n >> W;
    for(int i = 1; i <= n; i++) cin >> weight[i] >> value[i];

    for(int i = 1; i <= n; i++)
    {
        for(int j = 1; j <= W; j++)
        {
            if(j < weight[i]) 
                dp[i][j] = dp[i-1][j];
            else 
                // 注意这里与0-1背包的唯一区别：放入物品i后，状态从 dp[i][...] 转移，而不是 dp[i-1][...]
                dp[i][j] = max(dp[i-1][j], dp[i][j-weight[i]] + value[i]);
        }
    }
    
    cout << dp[n][W];
    return 0;
}
```
时间复杂度：$O(nW)$，空间复杂度：$O(nW)$。

#### 多重背包问题（朴素解法）

>给定 $n$ 种物品和一个最大承重为 $W$ 的背包。第 $i$ 种物品的重量为 $weight[i]$ ，价值为 $value[i]$ ，且最多有 $count[i]$ 件可用。问在不超过背包承重的前提下，最多能获得多少总价值

1. 阶段：以物品序号及背包当前容量为阶段  
2. 状态：$dp[i][j]$ 表示在前 $i$ 种物品中进行选择时，背包容量为 $j$ 的情况下，背包中的最大总价值  
3. 状态转移方程：枚举第 $i$ 种物品实际放入背包的件数 $k$（需满足 $0 \le k \le count[i]$ 且 $k \times weight[i] \le j$）。
在当前语境下即 $dp[i][j] = \max \limits_{0 \le k \le count[i]} \{ dp[i-1][j - k \times weight[i]] + k \times value[i] \}$
4. 初始条件：$dp[i][0]=0$ ， $dp[0][j] = 0$
5. 问题结果：$dp[n][W]$

多重背包的朴素解法本质上就是在 0-1 背包的基础上，为每种物品增加了一层数量选择的循环。同样可以使用一维滚动数组极大地优化空间复杂度（注意时间复杂度仍为 $O(n \times W \times count)$）：
- 状态转移方程：$dp[j] = \max(dp[j], dp[j - k \times weight[i]] + k \times value[i])$
- 遍历方向与顺序：最外层正序遍历物品 $i$；**中间层从大到小倒序遍历容量 $j$**（原因与 0-1 背包完全一致，必须确保调用的 $dp[j - k \times weight[i]]$ 是上一阶段的旧数据）；最内层正序枚举选择的件数 $k$。

```cpp
// 二维朴素多重背包dp
#include <bits/stdc++.h>
using namespace std;

int n, W;           // n件物品和背包容量W
int weight[100];    // 每件物品的重量数组
int value[100];     // 每件物品的价值数组
int cnt[100];       // 每件物品的件数
int dp[100][100] = {0}; // 二维dp表,这里的初始化隐含了边界条件

int main()
{
    cin >> n >> W;
    for(int i = 1; i <= n; i++) cin >> weight[i] >> value[i] >> cnt[i];

    for(int i = 1; i <= n; i++)
    {
        for(int j = 1; j <= W; j++)
        {
           if(j<weight[i]) dp[i][j] = dp[i-1][j];
           else for(int k=1;k<=cnt[i] && k*weight[i]<=j;k++)
           {
                // 注意左边项变成了 dp[i][j]，这样就能保留之前算出的最大值
                dp[i][j] = max(dp[i][j],dp[i-1][j-k*weight[i]]+k*value[i]);
           }
        }
    }
    
    cout << dp[n][W];
    return 0;
}
```
时间复杂度：$O(nW \times count)$，空间复杂度：$O(nW)$。

##### 二进制拆分优化

> 简单来说，就是把物品的数量 $count[i]$ 拆分成由 $1,2,4,...,2^m$ 件单个物品组成的大物品以及剩余不足2的整数次幂数量的物品，由 $count[i]-2^{\log_2{(count[i]+1)}-1}$ 件单个物品组成大物品

说白了就是用二进制对原数进行拆分，对于给定的物品数量 $C$，我们将其拆分为两部分：
- 二次幂部分：$1, 2, 4, 8 \dots 2^k$，满足 $1 + 2 + 4 + \dots + 2^k \le C$。
- 余数部分：$R = C - (1 + 2 + 4 + \dots + 2^k)$。
  
这两部分可以表示出小于$C$的所有整数，这样依赖就大大减少了枚举次数

```cpp
// 二进制拆分+滚动数组优化后的多重背包dp
#include <bits/stdc++.h>
using namespace std;

int n,W,wgt,val,c;    // 物品数量，背包容量，物品质量，物品价值，物品件数
int weight[50010];  // 储存划分后的重量的数组，长度要大于n*log_2{count}
int value[50010];   // 储存划分后的价值的数组，长度要大于n*log_2{count}
int dp[1010] = {0}; // dp表，长度略大于V即可

int main()
{
    cin >> n >> W;
    int index = 0;  // 新下标，相当于划分后的n

    // 录入数据并进行二进制拆分
    for(int i=1;i<=n;i++)
    {
        cin >> wgt >> val >> c;
        for(int k=1;k<=c;k*=2)  // 二次幂部分
        {
            c -= k;
            index++;
            weight[index] = wgt * k;
            value[index] = val * k;
        }
        if(c > 0)   // 余数部分
        {
            index++;
            weight[index] = wgt * c;
            value[index] = val * c;
        }
    }

    //DP
    for(int i=1;i<=index;i++)
    {
        for(int j=W;j>=weight[i];j--)
        {
            dp[j] = max(dp[j],dp[j-weight[i]]+value[i]);
        }
    }

    cout << dp[W];
    return 0;
}
```
时间复杂度：$O(nW \times \log_2{count})$，空间复杂度：$O(W)$。
